installation:
============
https://www.digitalocean.com/community/tutorials/how-to-install-and-configure-ansible-on-ubuntu-18-04

check if you can ssh from server to client (we need to add the rsa key first):
ssh matt@192.168.100.103

global inventory:
/etc/ansible/hosts

add to the file:

[servers]
bind-client1 ansible_ssh_host=192.168.100.103

PYTHON 2.7
ansible will fail if it doen't find python 2 on host

testing if it's ok
===================
ansible -m ping servers (previously configured in /etc/ansible/hosts )

ansible -m shell -a 'free -m' bind-client1

configuration:
==============
/etc/ansible/ansible.cfg

creating vaults:
===============
https://docs.ansible.com/ansible/latest/user_guide/playbooks_vault.html
https://www.cyberciti.biz/faq/how-to-set-and-use-sudo-password-for-ansible-vault/

ansible-vault create vars.yml
it will open a file, format:
my_var: myval

installing multiple packages with apt module
http://www.mydailytutorials.com/introduction-ansible-apt-package-repository/

run playbook:
ansible-playbook -i hosts ~/kube-cluster/kube-objects.yml

using vault variables:
======================
ansible-playbook -i hosts --ask-vault-pass kube-objects.yml

adding tags to roles and execute only tasks with that role:

ansible-playbook -i hosts --ask-vault-pass kube-objects.yml --tags "dashboard-be"
ansible-playbook -i hosts --ask-vault-pass kube-objects.yml --tags "security-app"
ansible-playbook -i hosts --vault-password-file=~/myfile kube-objects.yml --tags "security-app"

shutdown all vm:
ansible all -i cluster --become --ask-become-pass -m shell -a 'shutdown -h now'
